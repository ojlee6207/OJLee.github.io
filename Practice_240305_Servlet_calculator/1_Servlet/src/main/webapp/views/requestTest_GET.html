Get 방식으로 요청 후 응답 페이지 보기

특징1. GET방식으로 요청 시 URL의 Header영역에 데이터를 포함시켜 요청
=> 사용자가 입력한 값(데이터)들이 url에 노출됨
=> 보안 상 안전하지 않음 => 따라서, 로그인 or 회원가입 같은 경우 부적합함

특징2. Header영역에 전송하는 길이의 제한이 있음
=> 방대한 데이터를 전송하는 경우 초과된 데이터는 제외되고 전송됨
=> ex) 게시판 작성 => GET방식 부적합

특징3. url에 데이터가 노출되므로 즐겨찾기(북마크) 기능 가능 => url을 저장해두었다가 재사용(요청)
=> 검색 기능에 GET방식이 적합함
